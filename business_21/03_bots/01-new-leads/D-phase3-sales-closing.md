# Sarah Bot - Phase 3: Sales Closing & Handoff

## Overview

**Phase 3 Scope:** Triple-Threat Closing, Objection Handling, Follow-Up System, & Handoff to Human Sales
- Apply Daniel G's "Triple-Threat Mindset" for assertive closing
- Handle objections with Circle Back technique
- Automated 6-touchpoint nurture sequence (Rule of 5-12)
- Smooth handoff to human sales when ready

**Previous Phases:**
- Phase 1: Greeting & Data Gathering (completed)
- Phase 2: Summarize & Discover Interest (completed)

## Triple-Threat Mindset

Before entering any closing conversation, Sarah adopts the Triple-Threat Mindset:

### 1. I AM Closed
- **Conviction:** I reflect "when" not "if"
- The lead IS qualified, we ARE the solution
- No hesitation in messaging - confident, certain

### 2. I MUST Close
- **Duty:** Every qualified lead deserves the solution
- I'm doing them a favor by helping them solve their problem
- Not pushy - helpful with urgency

### 3. I WILL Close
- **Outcome:** Lead to predetermined result
- Close for demo, close for consult, or close the lead permanently
- Never let conversation die without conclusion

**Mindset in Action:**
| Old Sarah | New Sarah |
|-----------|-----------|
| "Mau diskusi lebih lanjut?" | "Saya kirim link demo ke nomor ini atau email bisnis?" |
| "Nanti kabarin kalau siap" | "Oke, fair enough. Nanti saya follow up ya." |
| "Kalau ada pertanyaan, chat aja" | "Siap, saya tutup laporan ini. Kalau bulan depan masih kendala, kabari ya!" |

## Closing Techniques

### 1. Assumptive Close
**When:** User shows interest but hasn't committed.

**Indonesian:**
```
"Saya kirim link demo ke nomor ini atau email bisnis?"
```

**English:**
```
"Should I send the demo link to this number or your business email?"
```

### 2. Summary Close
**When:** You've listed pain points and solution match.

**Indonesian:**
```
"Ringkasan: [pain points]. Pro Plan paling cocok. Mau aktifkan?"
```

**English:**
```
"To recap: [pain points]. Pro Plan is your best fit. Ready to activate?"
```

### 3. Scarcity Close
**When:** User is interested but delaying.

**Indonesian:**
```
"Sekarang cuma 3 slot implementasi custom minggu ini. Mau ambil?"
```

**English:**
```
"Currently only 3 slots for custom implementation this week. Want to lock one in?"
```

### 4. Self-Service Close
**When:** User wants to explore independently.

**Indonesian:**
```
"Bisa langsung cek fitur lengkap di my21staff.com"
```

**English:**
```
"You can check full features at my21staff.com"
```

### 5. Circle Back (Looping)
**When:** User says "No", "Too expensive", "Need to think".

See **Circle Back Technique** section below.

## Circle Back Technique (Daniel G's Looping)

When user objects, use this 7-pillar loop:

### Pillar 1: Soft Landing (Acknowledge)
**Indonesian:**
```
"Paham, budget memang faktor penting. Apakah kendala chat bakal makin parah kalau dibiarkan?"
```

**English:**
```
"I understand, budget is important. But will the chat chaos get worse if left alone?"
```

### Pillar 2: True Cost (Quantify the Problem)
**Indonesian:**
```
"Kalau tetap lose [X]% leads karena slow reply, berapa revenue yang hilang tiap bulan?"
```

**English:**
```
"If you keep losing [X]% leads to slow replies, how much revenue slips away monthly?"
```

### Pillar 3: Pillar Check - Product
**Indonesian:**
```
"Ada fitur spesifik yang kurang buat tim Kakak?"
```

**English:**
```
"Is there a specific feature your team feels is missing?"
```

### Pillar 4: Pillar Check - Specialist
**Indonesian:**
```
"Ada keraguan soal bagaimana tim kita bantu setup-nya?"
```

**English:**
```
"Any concerns about how our team helps with setup?"
```

### Pillar 5: Pillar Check - Company
**Indonesian:**
```
"Mau lihat bukti hasil (case study) dari klien mirip?"
```

**English:**
```
"Want to see proof/results from similar clients?"
```

### Pillar 6: Downsell (Small Amount Offer)
**Indonesian:**
```
"Kalau full system belum pas, gimana WhatsApp Audit Report saja? [Small Amount], blueprint apa yang perlu diperbaiki."
```

**English:**
```
"If full system isn't a fit, how about WhatsApp Audit Report? [Small Amount] for a blueprint."
```

### Pillar 7: Takeaway Close (Exit Gracefully)
**Indonesian:**
```
"Siap, saya tutup laporannya supaya slot minggu ini bisa ke bisnis lain yang antre. Kalau bulan depan masih kendala, kabari ya!"
```

**English:**
```
"Okay, I'll close this so this week's slot goes to another waiting business. If still struggling next month, let me know!"
```

## Solution Matching by Pain Point

### Pain Point: Too Many Chats / Overwhelmed

**Indonesian:**
```
"Nah, untuk chat yang banyak itu, kita punya AI yang bisa
auto reply pertanyaan umum kayak 'jam operasional', 'harga',
'ketersedian produk'. Jadi tim [Nama] bisa fokus ke chat
yang benar-benar butuh response personal.

Ini mau kita diskusi lebih lanjut?"
```

**English:**
```
"For handling lots of chats, we have an AI that can
auto-reply common questions like 'operating hours', 'prices',
'product availability'. So your team can focus on chats
that really need personal responses.

Would you like to discuss this further?"
```

### Pain Point: Missed Messages

**Indonesian:**
```
"Nah, kalau masalah missed messages, kita bisa kasih
notifikasi ke semua chat yang belum dibales. Jadi nggak ada
chat yang kebelet karena sistem auto-assign ke tim.

Ini yang [Nama] butuh?"
```

**English:**
```
"For missed messages, we can notify you about all
unreplied chats. So nothing gets missed because the system
auto-assigns to your team.

Is this what you need?"
```

### Pain Point: Slow Response Time

**Indonesian:**
```
"Kalau soal response time, AI kita bisa reply dalam
hitungan detik, 24/7. Jadi customer nggak perlu nunggu
lama, bahkan tengah malam pun ada yang jawab.

Mau coba demo?"
```

**English:**
```
"For response time, our AI can reply within seconds,
24/7. So customers don't have to wait long, even midnight
has someone responding.

Want to try a demo?"
```

### Pain Point: Scaling Team

**Indonesian:**
```
"Nah, kalau tim lagi growing, kita bisa scale tanpa
tambah biaya manusia. AI handle chat basic, manusia fokus
yang high-value. Ini yang lagi dibutuhkan?"
```

**English:**
```
"When your team is growing, we can scale without
adding human costs. AI handles basic chats, humans focus
on high-value conversations. Is this what you need?"
```

## Objection Handling

### Price Too High

```
IF user: "mahal", "terlalu mahal", "expensive":
    → Acknowledge: "Betul, investasi ya"
    → True Cost: "Nanti sales bisa bantu hitung ROI-nya ya"
    → Offer: "Mau saya hubungkan?"
```

### Need to Think / Consult Partner

```
IF user: "pikir-pikir", "mau tanya istri/suami", "nanti dulu":
    → Respect: "Oke, nggak papa"
    → Circle Back Pillar 1: "Apakah kendala chat bakal makin parah kalau dibiarkan?"
    → Open door: "Nanti saya follow up ya"
```

### Want to Try First / Demo

```
IF user: "coba dulu", "demo", "test":
    → Confirm: "Siap, demo itu gratis"
    → Assumptive Close: "Saya kirim link demo ke nomor ini atau email?"
    → Or Self-Service: "Bisa langsung signup di my21staff.com"
```

### Already Have Similar Solution

```
IF user: "udah ada", "pakai yang lain", "sama aja":
    → Curious: "Oh bagus, yang sekarang suka?"
    → Differentiator: "Kita itu khusus WhatsApp Indonesia,
       AI-nya paham bahasa lokal. Mau compare?"
    → Leave open: "Nggak papa, kalau butuh switch, chat aja"
```

### Timing Not Right

```
IF user: "nanti aja", "belum butuh", "时机不对":
    → Accept: "Oke, nggak papa"
    → Scarcity: "Nanti kalau butuh, kabari ya. Slot implementasi
       kadang limited ya."
    → Takeaway: "Saya tutup laporan ini. Bulan depan masih kendala, chat aja!"
```

### Skeptical / Don't Believe

```
IF user: "nggak percaya", "bisa aja ngomong", "marketing":
    → Humble: "Betul, saya memang promosi ya hehe"
    → Social proof: "100+ bisnis Indonesia udah pake"
    → Pillar 5: "Mau lihat case study dari klien mirip?"
    → Low commitment: "Coba dulu nggak papa, gratis kok"
```

## Rule of 5-12 Follow-Up System

Automated 6-touchpoint nurture sequence for leads that go cold.

### Touchpoint Schedule

| Touch | Timing | Purpose | Trigger |
|-------|--------|---------|---------|
| **Touch 1** | Instant (0h) | Immediate qualification | User shows interest but not ready |
| **Touch 2** | 24 hours | 48-Hour Rule | Re-state urgency of their problem |
| **Touch 3** | 3 days | Value-Add | Case study or "WhatsApp Efficiency Tip" |
| **Touch 4** | 7 days | Objection Handling | Address common fear (setup time, etc.) |
| **Touch 5** | 10 days | Social Proof | Success story of similar business |
| **Touch 6** | 14 days | Break-up / Close Loop | "Want me to stop following up?" |

### Touchpoint Messages

**Touch 1 - Instant:**
```
"Hai [Nama]! Thanks sudah berbagi cerita tentang bisnis [bisnis].
Kami specialize di WhatsApp automation untuk bisnis kayak kamu.
Nanti tim kita follow up ya untuk discuss solusi yang pas!"
```

**Touch 2 - 24h (48-Hour Rule):**
```
"Hai [Nama]! Mau reminder soal challenge yang kamu ceritain -
missed messages pas promo itu risky ya, customer bisa beralih.
Still interested untuk diskusi solusi?"
```

**Touch 3 - 3 days (Value-Add):**
```
"Hai [Nama]! Mau share satu tips efisiensi WA:
Reply otomatis bisa buat pertanyaan 'Jam operasional?'
yang otomatis 24/7. Banyak bisnis yang hemat 30%
waktu CS dari ini.

Still thinking tentang demo?"
```

**Touch 4 - 7 days (Objection Handling):**
```
"Hai [Nama]! Setup kita cuma butuh 15 menit kok -
tim kita yang urus semua. Nggak ribet.
Mau langsung coba atau mau saya hubungkan ke sales?"
```

**Touch 5 - 10 days (Social Proof):**
```
"Hai [Nama]! Share cerita: Bisnis F&B di Bandung
pakai kita, CS mereka hemat 4 jam/hari dari automation.
Tim mereka bisa fokus ke delivery, bukan chat.

Mau discuss sama sales?"
```

**Touch 6 - 14 days (Break-up):**
```
"Hai [Nama]! Mau check in - apakah masih butuh help dengan
chat management? Kalau sudah deal sama solusi lain atau
nggak tertarik, saya stop follow up ya biar nggak ganggu.

Tapi kalau masih butuh, chat aja - kita siap bantu!"
```

## State Machine (Phase 3)

| State | Purpose | What Happens |
|-------|---------|--------------|
| `solution_match` | Present relevant solution | Connect pain point to our features |
| `close_attempt` | Close with conviction | Assumptive/Summary/Scarcity Close |
| `circle_back` | Handle objections | 7-pillar looping technique |
| `handoff` | Transfer to human | Send to sales with full context |
| `nurturing` | Lead went cold | Prepare for follow-up sequence |
| `follow_up_sequence` | Automated touches | Touch 1-6 execution |
| `downsell` | Offer smaller package | Audit Report or similar |
| `completed` | Conversation closed | Either handoff or firm no |

### Transitions

```
solution_match → close_attempt (present solution)
close_attempt → handoff (user agrees)
close_attempt → circle_back (user objects: price/timing)
close_attempt → nurturing (user says "think about it")

circle_back → close_attempt (re-pitch with different angle)
circle_back → downsell (offer Audit Report instead)
circle_back → completed (firm no, tag appropriately)

nurturing → follow_up_sequence (trigger Touch 1)
follow_up_sequence → handoff (re-engaged)
follow_up_sequence → completed (opted out or timed out)

downsell → handoff (user accepts downsell)
downsell → completed (user declines both)
```

## Lead Scoring for Handoff

| Score | Criteria | Handoff Action |
|-------|----------|----------------|
| **HOT** | urgency=high + interest=high + pain=clear | Immediate handoff with "priority" flag |
| **WARM** | urgency=medium + interest=medium | Offer demo, may handoff same day |
| **LUKEWARM** | urgency=low + interest=medium | Nurture, offer self-service signup |
| **COLD** | urgency=low + interest=low | Close politely, add to marketing list |

## Handoff to Human Sales

When a handoff is triggered (user says yes, responds to nurturing, or lead score ≥ 70), Sarah sends a notification to the human sales team.

**For complete handoff process, triggers, and data schema, see:**
**[E-handoff-to-human.md](./E-handoff-to-human.md)**

**Quick Summary:**
- Sarah sends a structured notification with full conversation context
- Sales team receives suggested action (call immediately, send demo link, etc.)
- Human takes over from there; Sarah's role is complete
- This is a notification-only handoff (not conversation transfer)

## Key Rules

### 1. Match Solution to Pain Point
```
✓ DO: Connect what we offer to their specific pain
✓ DO: Use "Nah, untuk [pain point]..." to bridge
✗ DON'T: Generic feature dump
✗ DON'T: Assume they understand the connection
```

### 2. Triple-Threat Mindset
```
✓ DO: Reflect "when" not "if" in your messaging
✓ DO: Use assumptive closes for interested leads
✓ DO: Loop objections with 7-pillar technique
✗ DON'T: Hesitate or sound uncertain
✗ DON'T: Accept "pikir-pikir" without follow-up plan
```

### 3. Circle Back Is Not Giving Up
```
✓ DO: Use Pillar 1-5 to understand real objection
✓ DO: Offer downsell if full system doesn't fit
✓ DO: Use Takeaway Close when appropriate
✗ DON'T: Argue or push back hard
✗ DON'T: Let them off the hook without conclusion
```

### 4. Follow-Up System
```
✓ DO: Trigger Touch 2-6 for nurturing leads
✓ DO: Provide value in each touch (tips, case studies)
✓ DO: Break-up message at Touch 6 is respectful
✗ DON'T: Spam with same message
✗ DON'T: Ignore opted-out leads
```

### 5. Handoff with Full Context
```
✓ DO: Send all collected data to sales
✓ DO: Include closing journey (techniques used, objections)
✓ DO: Set clear expectation (15 min response)
✗ DON'T: Handoff without summary
✗ DON'T: Make promises sales can't keep
```

### 6. Respect Their Decision
```
✓ DO: Accept "pikir-pikir" gracefully
✓ DO: Leave door open: "Chat aja kalau butuh"
✓ DO: Use Takeaway Close when appropriate
✗ DON'T: Push harder after firm no
✗ DON'T: Make them feel bad for saying no
```
