4# Sarah Bot - Phase 2: Summarize & Discover Interest

## Overview

**Phase 2 Scope:** Reflective Summary + Pain Point Validation + Interest Discovery
- Mirror back what we learned (builds trust)
- Help them articulate their challenges
- Uncover why they're interested in us
- Build emotional connection before pitch

**Previous Phases:**
- Phase 1: Greeting & Data Gathering (completed)

**Future Phases:**
- Phase 3: Scoring & Classification
- Phase 4: Handoff & Closing

## Key Rules

### 1. Reflective Summary Builds Trust
```
✓ DO: Mirror back what they said (name, business, duration, key points)
✓ DO: Show you were actually listening
✗ DON'T: Just jump to next question without acknowledgment
```

### 2. Pain Points = Validation, Not Assumption
```
✓ DO: List pain points as "Dari yang X ceritakan..."
✓ DO: End with "Betul nggak?" or "Betul nggak ada yang lain?"
✓ DO: Let them confirm or add more
✗ DON'T: "Kamu pasti不爽 dengan X" (too presumptuous)
```

### 3. Interest Discovery = Open-Ended
```
✓ DO: Ask "Yang bikin tertarik itu apa?" (open)
✓ DO: Ask "Mana yang paling urgent?" (prioritization)
✓ DO: Accept "Cuma mau lihat2" (curiosity is valid)
✗ DON'T: Assume they have a problem just because they replied
```

### 4. Keep It Conversational
```
✓ DO: Use casual language: "Wah paham banget", "Hahaha iya"
✓ DO: React to their pain: empathy = connection
✓ DO: Keep it SHORT still - 2-3 sentences max
✗ DON'T: Long paragraphs explaining features
```

### 5. Validating Pain Point Keywords

| Keyword Category | Examples | Action |
|-----------------|----------|--------|
| Overwhelmed | "kewalahan", "banyak banget", "gila-gilaan" | Highlight this as main pain point |
| Missed messages | "miss", "kebelet", "nggak kebales" | Emphasize automation benefit |
| Slow response | "lambat", "terlambat", "nunggu lama" | Highlight speed benefit |
| Growth | "growing", "naik", "ekspansi" | Emphasize scalability |
| Curiosity | "mau lihat", "explore", "cari tahu" | Accept, don't push problem narrative |

## State Machine (Phase 2 Only)

| State | Purpose | What Happens |
|-------|---------|--------------|
| `summarize` | Reflective summary | Mirror back collected data |
| `validate_pain_points` | Highlight challenges | List pain points + validate |
| `discover_interest` | Find motivation | Open-ended interest question |
| `confirm_priority` | Prioritize needs | What matters most? |

### Transitions

```
summarize → validate_pain_points (after summary sent)
validate_pain_points → discover_interest (pain points validated)
discover_interest → confirm_priority (interest mentioned)
confirm_priority → [PHASE 3: handoff]
```

### Message Flow

```
1. "Oke [Nama], jadi [bisnis] di [domisili], udah [X tahun] ya..."
   → User acknowledges

2. "Dari yang [Nama] ceritakan, challenge-nya kan: [list] Betul nggak?"
   → User confirms/adds

3. "Layanan kita itu [relevant benefit]. Ini yang [Nama] butuh atau
   ada yang lain?"
   → User responds

4. "Oke, jadi [summarize their priority]. Data sudah lengkap,
   nanti tim kita yang follow up ya."
   → Move to Phase 3: Handoff
```

## Language Detection (Phase 2)

Continue using same detection from Phase 1. If user switches language mid-conversation, follow their preference.

```typescript
function getPhase2Messages(language: 'id' | 'en', data: Phase1Data): {
  summarize: string;
  validatePain: string;
  discoverInterest: string;
} {
  if (language === 'en') {
    return {
      summarize: `Okay ${data.name}, so you run a ${data.business_type} in ${data.domisili}, been at it for ${data.duration || 'a while'}. Thanks for sharing!`,
      validatePain: `From what you've shared, the challenges are: ${data.painPoints.join(', ')}. Is that right? Anything else?`,
      discoverInterest: `Our service is an AI that handles customer chats, auto-replies, and assigns to your team. Is this what you need, or something else?`
    };
  }

  return {
    summarize: `Oke ${data.name}, jadi ${data.business_type} yang berdomisili di ${data.domisili}, udah ${data.duration || 'lama'} ya. Thanks sudah sharing!`,
    validatePain: `Dari yang ${data.name} ceritakan, challenge-nya kan: ${data.painPoints.join(', ')}. Betul nggak? Ada yang lain lagi?`,
    discoverInterest: `Layanan kita itu basically AI yang bantu handle chat, auto reply, assign ke tim. Ini yang ${data.name} butuh atau ada yang lain?`
  };
}
```

## Edge Cases

### User Doesn't Confirm Pain Points

```
IF user says "Nggak" or adds different pain points:
    → Update pain_points array with their correction
    → Re-validate: "Oh jadi [new pain point] ya, betul?"
    → Continue to interest discovery
```

### User Has No Pain Points (Just Curious)

```
IF user: "Cuma mau lihat2", "Mau explore":
    → Accept curiosity, don't push problem narrative
    → Ask: "Ada yang bikin kamu penasaran dari layanan kita?"
    → Provide brief feature overview
    → Offer to connect to sales for demo
    → Tag: "curious_browser"
```

### User Wants All Features

```
IF user says "Semua butuh!" or lists many needs:
    → Help prioritize: "Dari semua itu, mana yang paling urgent
       harus solved bulan ini?"
    → Focus on 1-2 top priorities
    → Mark urgency as high
```

### User Asks Feature Details

```
IF user asks specific questions: "Bisa nggak X?", " gimana Y?":
    → Give brief answer (1-2 sentences)
    → Return to interest discovery
    → Offer sales handoff for deep-dive
```
