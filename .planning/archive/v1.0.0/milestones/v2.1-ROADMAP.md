## Milestone Summary

**Key Decisions:**
- Use Resend HTTP API instead of SMTP (solved DNS resolution issues)
- TanStack Query for client-side caching (instant navigation feel)
- Separate business/ folder for non-code assets
- 4-stage support ticket workflow for trust-building

**Issues Resolved:**
- P0 SMTP DNS resolution issue (switched to Resend)
- P0 RLS policy gaps (added permission enforcement)
- Performance issues (added loading skeletons + client caching)

**Known Issues Deferred:**
- Forgot password still uses Supabase email
- Resend/delete invitation has auth bug

**Technical Debt:**
- In-memory rate limiting won't scale multi-instance (noted for v2.2)
